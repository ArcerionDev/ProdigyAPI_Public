Dungeon endpoint

The Dungeon endpoint gives info on dungeons in Prodigy, in which you need to do a certain task before exiting (e.g. Archives, Earth Tower)

Endpoint:

http://localhost:4000/v1/dungeon

Example request:

fetch('http://localhost:4000/v1/dungeon/').then(response => response.json()).then(data => {console.log(data)})

Object properties:

dungeon[dungeonID].ID
dungeon[dungeonID].assetID
dungeon[dungeonID].type
dungeon[dungeonID].gender
dungeon[dungeonID].data
dungeon[dungeonID].metadata
dungeon[dungeonID].name

Data properties:

dungeon[dungeonID].data.version
dungeon[dungeonID].data.name
dungeon[dungeonID].data.seed
dungeon[dungeonID].data.gridWidth
dungeon[dungeonID].data.gridHeight
dungeon[dungeonID].data.minCritPathLength
dungeon[dungeonID].data.maxCritPathLength
dungeon[dungeonID].data.monsterIDs
dungeon[dungeonID].data.minEncounters
dungeon[dungeonID].data.maxEncounters
dungeon[dungeonID].data.monsterDrops
dungeon[dungeonID].data.goldenPages
dungeon[dungeonID].data.minPages
dungeon[dungeonID].data.maxPages
dungeon[dungeonID].data.battleDifficulty
dungeon[dungeonID].data.extraPaths
dungeon[dungeonID].data.monsterRooms
dungeon[dungeonID].data.normalRooms
dungeon[dungeonID].data.endRoom

Get more specific results using the Dungeon ID in the API endpoint.

http://localhost:4000/v1/dungeon/ID/

Example request:

fetch('http://localhost:4000/v1/dungeon/0/').then(response => response.json()).then(data => {console.log(data)})
